# Serial Communication (UART, I2C, SPI) — Fundamental Reference

## What It Is

Serial communication sends data **one bit at a time** over one or two wires. Instead of needing one wire per bit (parallel), you send bits in sequence. Slower per-pulse, but far fewer wires.

The three main protocols for hobby electronics:

| Protocol | Wires | Speed | Devices per bus | Distance | Best for |
|----------|-------|-------|-----------------|----------|----------|
| UART | 2 | Up to 2 Mbps | 2 (point-to-point) | ~5m | GPS, Bluetooth, debug console |
| I2C | 2 | Up to 3.4 MHz | Up to 127 | ~1m | Sensors, RTC, OLED, ADC |
| SPI | 4 | Up to 80 MHz | Many (per chip-select) | ~10cm | Displays, SD cards, shift registers |

## UART (Universal Asynchronous Receiver-Transmitter)

### Wiring

```
Device 1          Device 2
  TX ──────────── RX
  RX ──────────── TX
  GND ─────────── GND
```

TX of one goes to RX of the other. **Cross them over** — this is the most common mistake.

### How It Works

No clock wire. Both devices agree on the speed (baud rate) beforehand:
- 9600 baud — slow, reliable, default for most modules
- 115200 baud — fast, common for debug consoles
- 4800 baud — GPS modules

Data is sent as packets: start bit → 8 data bits → optional parity → 1 or 2 stop bits.

### Voltage Levels

UART signals at the microcontroller's logic voltage (3.3V or 5V). If your GPS module is 3.3V and your Arduino is 5V, you need level shifting (or a voltage divider on the TX line).

### Common Uses

- Serial monitor / debug console
- GPS modules (NEO-6M, NEO-8M)
- Bluetooth modules (HC-05, HC-06)
- Some sensors (PM2.5, CO2)
- Programming/bootloading many microcontrollers

## I2C (Inter-Integrated Circuit)

### Wiring

```
Master          SDA ───── SDA    Slave
                SCL ───── SCL
               VCC ───── VCC
               GND ───── GND
              ┌─[4.7kΩ]─┐
              └─[4.7kΩ]─┘
              Pull-up resistors to VCC
```

Both wires need **pull-up resistors** (typically 4.7kΩ to VCC). Many modules have them built-in.

### How It Works

- **SDA** (Serial Data) — carries the data
- **SCL** (Serial Clock) — carries the clock signal generated by the master
- Each device has a unique **address** (7-bit: 0x00–0x7F)
- Master initiates communication, slaves respond when addressed
- Multiple masters possible (but rarely used in hobby projects)

### Pull-up Resistor Values

| Bus speed | Resistor value | Wire length |
|-----------|---------------|-------------|
| 100 kHz (standard) | 4.7kΩ | < 50cm |
| 400 kHz (fast) | 2.2kΩ–4.7kΩ | < 30cm |
| 1 MHz (fast+) | 1kΩ–2.2kΩ | < 10cm |

Too high = slow edges (signal distortion). Too low = too much current (hot resistors, possible damage).

### Common Addresses

| Device | Typical Address |
|--------|----------------|
| OLED SSD1306 | 0x3C (or 0x3D) |
| LCD 1602 I2C | 0x27 (or 0x3F) |
| BMP280/BME280 | 0x76 (or 0x77) |
| MPU6050 | 0x68 |
| DS3231 RTC | 0x68 |
| AT24C32 EEPROM | 0x50 |

### Common Uses

- OLED / LCD displays
- Temperature/humidity/pressure sensors
- Real-time clocks
- External ADC/DAC chips
- EEPROM memory

## SPI (Serial Peripheral Interface)

### Wiring

```
Master          MOSI ──── MOSI    Slave
                MISO ──── MISO
                SCK ───── SCK
                CS ────── CS
               VCC ───── VCC
               GND ───── GND
```

- **MOSI** (Master Out, Slave In) — data from master to slave
- **MISO** (Master In, Slave Out) — data from slave to master
- **SCK** (Serial Clock) — clock generated by master
- **CS** (Chip Select) — one per slave, active LOW

### Multiple Slaves

Each slave needs its own CS pin. The master pulls a slave's CS low to talk to it, leaving other CS pins high:

```
Master          MOSI ──── MOSI  Slave 1
                MISO ──── MISO
                SCK ───── SCK
                CS1 ───── CS
                CS2 ────  CS    Slave 2
                CS3 ────  CS    Slave 3
```

### Speed

SPI is the fastest of the three. Common speeds:
- 1–10 MHz — typical for most sensors/displays
- 20–40 MHz — fast displays, SD cards
- 80 MHz — maximum on many microcontrollers

Higher speed = shorter wires needed. Above 10 MHz, keep wires under 10cm.

### Common Uses

- SD cards
- High-speed displays (ILI9341 TFT, ST7789)
- RF modules (nRF24L01, LoRa)
- Shift registers (74HC595 — though that uses a SPI-like protocol)
- DAC chips

## When to Use What

| Situation | Protocol |
|-----------|----------|
| Talk to one sensor | I2C or UART |
| Debug logging to PC | UART |
| Connect many sensors | I2C (2 wires for many devices) |
| Fast display / SD card | SPI |
| Long distance (meters) | UART (with RS232/RS485) |
| GPS module | UART |
| Bluetooth module | UART |
| Simple 2-device communication | UART |
| 5+ devices with minimal wiring | I2C |

## Quick Reference

- **UART:** TX → RX, RX → TX, GND → GND. Set same baud rate both sides
- **I2C:** SDA + SCL + pull-up resistors (4.7kΩ). Unique address per device
- **SPI:** MOSI → MOSI, MISO → MISO, SCK → SCK, separate CS per slave
- **Level shifting:** Needed when mixing 3.3V and 5V devices on the same bus
- **Shared ground:** ALL protocols need GND connected between devices
